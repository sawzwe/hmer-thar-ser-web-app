Phase 1 · Foundation

Route Groups & Auth Guards

Create (vendor) and (admin) layout.tsx with UserFactory.fromSupabase, redirect if wrong type. Build VendorShell and AdminShell sidebars.

Phase 2 · Vendor CMS

Vendor Dashboard Pages

Dashboard, restaurants list/editor, deals, slots, bookings, claim flow. All scoped to user.restaurantIds. Always call user.ownsRestaurant(params.id) on [id] pages.

Phase 3 · Admin Panel

Admin Dashboard Pages

Overview, vendors (verify), users, restaurants, bookings, reviews. Gate destructive actions with user.isSuperAdmin().

Phase 4 · Security

API Route Guards

requireVendorOwner(restaurantId) and requireAdmin(). Override restaurant_id from URL, never from body. RLS policies for vendor-scoped tables.

Phase 5 · Polish

Navigation & UX

Add Vendor Dashboard and Admin Panel links to AppShell user menu. Middleware redirect for /vendor and /admin when unauthenticated.

Cursor Agent Prompts

5 Copy-paste Prompts

Run in order: (1) Route groups + shells, (2) Vendor pages, (3) Admin pages, (4) API routes, (5) RLS + nav + middleware.
